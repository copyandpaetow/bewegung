Target

HTMLElement | Element | HTMLElement[] | Element[] | NodeListOf<Element> | HTMLCollection | string;

Can be one or multiple elements or any valid css selector.

height and max-height: while it's technically possible to animate these properties, it results in non-performant animations(opens in a new tab) since they cause a layout recalculation at every frame.

visibility: Can't be animated. CSS transitions work by calculating intermediate amounts between two values. As visibility changes between visible/hidden, there are no possible intermediate values to extrapolate. Some "hacks"(opens in a new tab) allow CSS transitions using visiblity and opacity using transition-delay, but this technique is better suited for fade-in/fade-out effects rather tha opening/collapsing effects (which is most likely the effect you're aiming for an accordion).

This technique will have some drawbacks since it will not take the height of the content to be animated into consideration. Another major downside, is that the animation will not be very performant since it is using max-height, triggering Layout, Paint, and Composite(opens in a new tab) events on every frame.
